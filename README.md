Azure Labs
Hands-on Azure administration and cloud security engineering work, documented as a running portfolio. Each lab covers a specific area of Azure — identity, governance, networking, storage, compute, and monitoring — and is built around a realistic business scenario with the security considerations that shaped the design.
About
I'm a cybersecurity engineer with ~6 years of federal experience (TS cleared, CISSP, GICSP) focused on cloud security engineering in Azure environments. This repo documents the technical work behind that focus: identity and access management, governance and policy, network security, secure storage and compute, and monitoring and detection.
Repo structure
Every lab is a self-contained folder with:

A README walking through the business scenario, the architecture, what was built, and the security considerations behind the design
Screenshots from the Azure portal documenting the implementation
Supporting artifacts where relevant (ARM/Bicep templates, KQL queries, policy JSON, PowerShell scripts)

Resource naming follows a consistent enterprise pattern across every lab so the environment reads as a coherent deployment rather than disconnected exercises. See naming-conventions.md for the full pattern.
